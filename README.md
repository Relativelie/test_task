## first task
1. Перейти в папку, содержащую скрипт:
``cd  scripts/firstTask ``
2. Установить зависимости:
``npm install ``
3. Запустить скрипт:
``node firstTask.js ``

* Описание решения: рекурсивно находим все текстовые файлы в корневой папке, сортируем их по имени и далее с помощью конкатенации склеиваем сожержимое папок в отдельный файл

## third task
1. Перейти в папку, содержащую проект:
``cd  scripts/third-task ``
2. Установить зависимости:
``npm install ``
3. Зайти на страницу https://cors-anywhere.herokuapp.com/corsdemo разблокировать доступ к демо-серверу. Иначе при попытке дсотучатсья до АПИ будет ошибка CORS
4. Запустить проект
``npm start``

* Описание: что успела сделать: поиск по списку задач, вывод списка задач, фильтрация по статусу(без каких лио стилей)
Фильтрация была сделана на стороне клиента, так как было не понятно как запрос отправить(пробовала в from/to вставлять timestamp, просто даты, но возвращался пустой список всегда)
Из-за того, что фильтрация сделана на фронте, а поиск по наименованию через запрос на сервер, имеется один баг: если сначала отфильтровать, а потом вводить в поле поиска - то поиск по отфильтрованному списку не будет работать. А если фильтровать после поиска - то работает корректно
